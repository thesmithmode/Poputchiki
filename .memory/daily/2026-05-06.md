**Context:** Poputchiki — рефактор TASK-102 (anti-bot middleware) + починка падающего CI

**Key Exchanges:**

- Решение: вместо `.use("/", antiBot)` (применяется ко всем методам) — переход на `app.post("/", antiBot(sql), handler)` chain pattern в Hono
- CI упал из-за lint: `resolve!` → `resolve?.`, `console.log` → `console.info`
- TS7030: `MiddlewareHandler` требует явного return; TS2454: `let resolve` без инициализации → `let resolve: (() => void) | undefined`

**Decisions Made:**
- Anti-bot middleware применяется только к ride-requests через Hono multiple handler chain, не через `.use()` (чтобы не ломать GET и не требовать SQL в GET тестах)
- `/api/ride-requests POST` (`/:id/request`) — anti-bot не применяется (разная семантика: flood защита для создания поездок, не откликов)

**Lessons Learned:**
- Hono `.use("/", middleware)` применяется ко ВСЕМ HTTP методам на пути, включая GET — для POST-only protection использовать `app.post("/", middleware, handler)` chain
- `vi.fn()` без реализации возвращает `undefined` → деструктуризация падает; middleware что делает SQL должен либо мокироваться в тестах, либо регистрироваться только там где вызывается
- `let resolve: (() => void) | undefined` обязательно для TS strict при использовании переменной в closure до присвоения
