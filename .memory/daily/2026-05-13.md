**Context:** Серия деплоев продолжает падать — диагностика и фиксы в прямом эфире

**Lessons Learned:**
- `pg_hba.conf` не монтировался явно → дефолт разрешает только localhost (127.0.0.1, ::1) → Docker-сеть 172.20.0.x получает "no pg_hba.conf entry". Решение: создать `infra/postgres/pg_hba.conf` с `host all all 172.16.0.0/12 scram-sha-256`, добавить `hba_file = '/etc/postgresql/pg_hba.conf'` в `postgresql.conf`, примонтировать volume
- `POSTGRES_DB` env срабатывает ТОЛЬКО при первой инициализации пустого тома. Если том уже существует — БД не создаётся. При первом деплое с готовым томом нужно создавать БД вручную: `psql -U $user -d postgres -c "CREATE DATABASE ..."`
- Хардкод имени БД в init SQL — антипаттерн: `GRANT CONNECT ON DATABASE poputchiki TO role` ломается при `POSTGRES_DB=poputchiki_test`. Правильно: `DO $$ BEGIN EXECUTE format('GRANT CONNECT ON DATABASE %I TO poputchiki_app', current_database()); END $$;`
- Init-скрипты `/docker-entrypoint-initdb.d/` запускаются один раз (пустой том). Изменения не применяются повторно — применять руками через psql
- deploy.yml триггерится только на `main`. Merge dev→main происходит только по явной команде пользователя
- SSH ключ для сервера: `~/.ssh/claude_racknerd_120`, пользователь `root@192.3.12.148`
- TG-уведомления о падении деплоя идут ТОЛЬКО на `ADMIN_TG_CHAT_ID` — обычные пользователи не видят
