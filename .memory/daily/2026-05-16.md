### Session (12:31)

**Context:** Завершение секьюрити-рефакторинга auth-системы — замена cookie tg_uid на sess_bind (HMAC-привязка к JWT jti)

**Key Exchanges:**
- Восстановление контекста после /compact — пользователь дал подробный промежуточный итог с 6 прод-файлами и списком оставшихся тест-файлов
- ~40 тест-файлов обновлены batch-скриптами + ручными правками для edge cases (разные token-переменные: driverToken, otherToken)

**Decisions Made:**
- `tg_uid` cookie полностью заменён на `sess_bind=HMAC(jwtSecret, jti)` — привязка сессии к конкретному JWT, не к Telegram ID
- ACCESS_TTL сокращён с 24h до 15min
- JWT_SECRET минимум увеличен с 16 до 32 символов
- NONCE_TTL увеличен с 10min до 1h (закрытие 50-минутного replay window)
- `verifyInitData` — направленная проверка freshness (age < -30 || age > MAX) вместо Math.abs

**Lessons Learned:**
- Batch sed-замена в тестах требует предварительной проверки edge cases (файлы где token-переменная не `token` а `driverToken`/`otherToken`)
- `authHeaders(user, token)` helper-паттерн требовал обновления сигнатуры — user param больше не нужен для cookie
- PowerShell batch-скрипты эффективны для механических замен в 20+ файлах, но inline cookie-паттерны (не через helper) требуют отдельной обработки
