## Key Points

- Мини-баннер над карточками поездок показывает возраст закэшированных данных
- Текст «обновлено N секунд назад» обновляется каждую секунду локально (без лишних запросов)
- Кликабельная иконка ↻ вызывает `invalidateQueries` — принудительный рефетч
- Появляется только когда данные старше порогового значения (например 10 с), при свежих данных скрыта

## Details

Индикатор реализуется через `dataUpdatedAt` из TanStack Query — timestamp последнего успешного fetch, хранящийся в кэше. Компонент считает разницу `Date.now() - dataUpdatedAt` в `useEffect` с `setInterval(1000)`, обновляя локальный счётчик каждую секунду. Это не создаёт сетевых запросов — только UI-обновление. Пороговая видимость (показывать только при `age > threshold`) убирает визуальный шум: при активном использовании данные всегда свежие и баннер не нужен.

Паттерн решает проблему [[stale-ride-feed-ux-trap]]: даже при `staleTime: 20_000` пользователь может смотреть на данные возрастом 19 секунд. Индикатор даёт ему agency — он видит, что данные могут быть устаревшими, и может нажать ↻ перед тем как записаться в поездку. Ручной рефреш через `queryClient.invalidateQueries` инвалидирует кэш и вызывает немедленный background fetch, после которого `dataUpdatedAt` обновляется и баннер скрывается.
