# Лента+карта симметрия — общий хедер с пресетами фильтров

## Суть

FeedScreen и MapScreen объединены под **симметричным верхним хедером**: одинаковые фильтры (откуда/куда/цена/дата) + кнопка "Избранное" (пресеты) видны на обоих вкладках. Контент вкладок разный: лента показывает список RideCard, карта — маркеры Leaflet. BottomTabBar: «Лента» + «Карта» (раздельные вкладки, не один экран).

## История решения

**Попытка 1 (session 22:40):** единый `RidesScreen` с переключателем лента/карта внутри — убрать «Карта» из BottomTabBar. Реализовано в рамках TASK-083/TASK-087.

**Откат (session 23:48):** UI-регрессия — карта ломалась после рефакторинга BottomTabBar (Leaflet не получал размер при hidden-состоянии). Решение: **вернуть «Карта» в BottomTabBar**, но синхронизировать хедер+фильтры между вкладками.

**Финал (commit `b2ae4f1`):** обе вкладки рендерят один и тот же `<RidesHeader>` с фильтрами — симметрия достигнута без объединения в один экран.

## Почему Leaflet ломается при скрытии

Leaflet вычисляет размер контейнера при монтировании. Если контейнер `display:none` или `visibility:hidden` в момент mount — карта получает нулевой размер и не восстанавливается корректно при показе. Решение: держать `<MapScreen>` смонтированным (CSS tab visibility), либо вызывать `map.invalidateSize()` при показе.
