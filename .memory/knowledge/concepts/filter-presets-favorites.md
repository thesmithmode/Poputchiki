# «Избранное» = пресеты фильтра, не лайки поездок

## Суть

В Poputchiki вкладка/экран «Избранное» хранит **сохранённые пресеты фильтра** (откуда → куда + цена + дата), а **не** список понравившихся/лайкнутых поездок. Смешение двух смыслов — продуктовая ошибка, приводящая к неверному API и неверной UX.

## Обоснование

Пользователи ЖК Царёво ездят по одному и тому же маршруту регулярно (дом ↔ работа/магазин). «Избранное» — быстрый доступ к частым фильтрам, одним тапом восстанавливающий привычный запрос. Лайки поездок не имеют смысла в контексте попутчиков — поездка либо актуальна, либо нет.

## Что переработать

- `FavoritesScreen` — показывать список сохранённых пресетов, применять пресет по тапу
- API `/favorites` — CRUD пресетов (name, from_address, to_address, max_price), не ride_ids
- DB таблица `favorite_presets` вместо `favorite_rides` (или `ride_likes`)
- Кнопка «Избранное» в хедере лент — открывает список пресетов для быстрого применения
